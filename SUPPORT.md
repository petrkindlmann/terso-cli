# Support

## Where to ask

| Question type | Where |
|---|---|
| Bug — something broken | [GitHub Issues](https://github.com/petrkindlmann/terso-cli/issues/new?template=bug.yml) |
| Feature request | [GitHub Issues](https://github.com/petrkindlmann/terso-cli/issues/new?template=feature.yml) |
| Usage question / discussion | [GitHub Discussions](https://github.com/petrkindlmann/terso-cli/discussions) |
| Security disclosure | security@omnus.dev — see [SECURITY.md](SECURITY.md) |
| Omnus account / billing | support@omnus.dev |

Please check the [troubleshooting guide](docs/troubleshooting.md) and existing
issues before opening a new one.

## Response time

Any non-spam issue gets an acknowledging reply within **72 hours** during the
8-week launch window around v1.0. Outside that window we aim for a one-week
response, with longer waits possible on weekends and holidays.

This is a small-team OSS project, not a paid product. We don't promise a
resolution timeline — only that we'll engage. If something is urgent for your
team, please say so in the issue.

## Public roadmap

`v1.0-launch` and `v1.1-omnus-connected` milestones are tracked at
[GitHub Milestones](https://github.com/petrkindlmann/terso-cli/milestones).
Open issues are labeled with the milestone they're targeted at.
